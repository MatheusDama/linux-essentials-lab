
O **Comand Line Interface Prompt** (ou Shell Prompt) é a grande caixa de texto que nos dá as boas-vindas. Para não me perder, adotei um panteão baseada na divisão dos dois-pontos (`:`):

Chamados de "Antes-de-Dois-Pontos" e "Após-Dois-Pontos" (escutei um aleluia irmão?)

**`sysadmin@localhost:~$`**

*   **Antes-de-Dois-Pontos (`sysadmin@localhost`):** Indica **Quem** está logado e **Onde**. No caso, o usuário `sysadmin` na máquina local (`localhost`).
*   **Após-Dois-Pontos (`~$`):** Indica **Onde no sistema** eu estou pisando e **Qual o meu poder (privilégio)**.

### Alguns Privilégios
* `$` Usuário comum
* `#` Usuário -root- ou

### Linux é um KERNEL (Que nome de mágia é isso?)

* **Linux significa *Kernel***, a qual é um Sistema Central de Controle para ***TUDO*** que acontece no computador.

  Linux é a combinação de ***SOFTWARES*** chamados **GNU/Linux**, definido pelo sistema **GNU**, que provem o acesso aberto de muitos comandos da base **UNIX**, e o *Linux Kernel*, que é o núcleo do sistema operacional.

📜 Como funciona essa divisão
* **Código-fonte:** É o texto escrito por programadores em linguagens como C++, Python ou Java. Ele possui comentários e nomes de variáveis fáceis de entender. É o segredo comercial da empresa.

* **Código de máquina:** É o resultado da tradução (compilação) do código-fonte em linguagem de computador (zeros e uns). O processador entende perfeitamente, mas para um ser humano é quase impossível de ler diretamente
